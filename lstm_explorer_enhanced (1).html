// INTRODUCTION

What is an LSTM?

A Long Short-Term Memory network is a special kind of Recurrent Neural Network (RNN) designed to learn long-range dependencies in sequential data. Proposed by Hochreiter & Schmidhuber in 1997, LSTMs solved a fundamental problem that plagued vanilla RNNs: the inability to remember information over many time steps.

Unlike a standard feedforward network, an LSTM processes data one step at a time — maintaining a hidden state (short-term memory) and a cell state (long-term memory) that flow from one timestep to the next, updated by a system of learnable gates.

Key intuition: Think of the cell state as a conveyor belt running through the network. Information can be added or removed from it via the forget, input, and output gates — each of which learns when to open or close based on the current input and previous hidden state.
// MOTIVATION

01 The Vanishing Gradient Problem

Standard RNNs struggle to learn from sequences longer than ~10 steps because gradients vanish (shrink toward zero) or explode (grow unboundedly) as they're backpropagated through time. This makes it impossible for the network to learn that a word at position 1 matters for a prediction at position 100.

The root cause: at each time step the gradient is multiplied by the same weight matrix. If the largest eigenvalue of that matrix is <1, gradients vanish; if >1, they explode.

Example: Consider "The cat, which sat on the warm mat by the window all afternoon in the sun, was …" A vanilla RNN loses the subject "cat" by the time it predicts the verb. An LSTM retains it.

LSTMs address this with an additive update to the cell state rather than a multiplicative one. Gradients can flow back through time without vanishing, as long as the forget gate stays open.

// ARCHITECTURE

02 The Cell State

The cell state Ct is the long-term memory of the LSTM. It runs as a horizontal "highway" through every time step, modified only by small, targeted operations — not by a full matrix multiplication each step. This is what allows gradients to flow and information to persist.

// Cell state update (the core equation)
Ct = ft ⊙ Ct-1 + it ⊙ tanh(WC · [ht-1, xt] + bC)
// ⊙ = element-wise multiplication
// f = forget gate, i = input gate

The first term ft ⊙ Ct-1 selectively erases old memory. The second term it ⊙ C̃t writes new candidate content. Their sum is the updated cell state carried forward.

// WORKED EXAMPLE

★ Step-by-Step: Processing "CAT"

Let's trace an LSTM processing the word "CAT" character by character with a hidden size of 2. We use simplified weights to illustrate the math clearly.

// Setup: Input = "C", "A", "T" | Hidden size = 2
INIT
Initialize h₀ = [0, 0] (hidden state) and C₀ = [0, 0] (cell state). All weights are pre-trained small random values near zero. Forget gate bias initialized to 1 (encourage memory).
t=1: 'C'
Concatenate input + hidden: [x_C, h₀] = [1,0,0, 0,0] (one-hot 'C' + zero hidden).
Forget gate: fₜ = σ(Wf·[x,h] + bf) ≈ [0.73, 0.69] — retains ~70% of nothing (irrelevant at t=1).
Input gate: iₜ = σ(Wi·[x,h] + bi) ≈ [0.52, 0.48] — about half the candidate accepted.
Candidate: C̃ₜ = tanh(Wc·[x,h] + bc) ≈ [0.31, -0.12] — new content from 'C'.
Cell update: C₁ = f⊙C₀ + i⊙C̃ ≈ [0.16, -0.06] — small first memory imprint.
Output: hₜ = oₜ⊙tanh(C₁) ≈ [0.08, -0.03] — initial hidden state after 'C'.
t=2: 'A'
Concatenate: [x_A, h₁] — 'A' one-hot plus prior hidden state [0.08, -0.03].
The forget gate now sees context from 'C'. It might partially erase the 'C' imprint if 'A' is dissimilar.
Input gate writes new 'A' information into the cell state, blended with what 'C' left.
Result: C₂ is a mixture of 'C' memory and 'A' content. Hidden h₂ reflects both characters.
t=3: 'T'
At the final step, the LSTM processes 'T' with full context of "CA" in h₂, C₂.
The output gate filters the combined cell state to produce h₃, which encodes the full sequence "CAT".
This final hidden state h₃ would be used for prediction — e.g., the next character or a classification label.
Key takeaway: At every step, the gates decide what to keep, what to write, and what to output. After 3 steps, the LSTM's hidden state summarizes the entire sequence "CAT" in a fixed-size vector — regardless of how long the original sequence was.
// GATES

03 The Forget Gate

The forget gate decides what fraction of the previous cell state to keep. It outputs a vector of values between 0 and 1 — 0 meaning "erase completely", 1 meaning "keep entirely".

ft = σ(Wf · [ht-1, xt] + bf)
// σ = sigmoid → output in (0,1)
Example: In language modelling, the forget gate might reset gender information about a subject ("she") when a new subject appears, so the network doesn't erroneously apply feminine pronouns later.
// GATES

04 The Input Gate

The input gate has two parts working together: a sigmoid layer deciding which values to update, and a tanh layer creating a vector of candidate values to potentially add.

it = σ(Wi · [ht-1, xt] + bi) // how much to update
C̃t = tanh(WC · [ht-1, xt] + bC) // candidate values

The result it ⊙ C̃t is added to the cell state — writing new information into long-term memory.

// GATES

05 The Output Gate

The output gate determines what the next hidden state ht will be — i.e., what the LSTM exposes to the next layer or produces as output. It reads from the updated cell state and filters it.

ot = σ(Wo · [ht-1, xt] + bo)
ht = ot ⊙ tanh(Ct)
Forget Gate
σ(Wf·[h,x]+bf)
Erases irrelevant past memory. Controls how much of Ct-1 survives.
Input Gate
σ(Wi·[h,x]+bi)
Decides which new information is written into the cell state.
Candidate Gate
tanh(WC·[h,x]+bC)
Creates new candidate values that could be added to the cell state.
Output Gate
σ(Wo·[h,x]+bo)
Filters the cell state to produce the hidden state output ht.
// PURPOSE & APPLICATIONS

✦ Purpose & Where LSTMs Are Used

LSTMs were designed to solve a single critical problem: learning dependencies across long time spans. Their purpose is to be a trainable, differentiable memory unit that selectively reads, writes, and erases information based on what the network learns is important.

Natural Language Processing
Sentiment analysis, machine translation, named entity recognition, text generation. LSTMs powered Google Translate before Transformers.
Speech & Audio
Speech-to-text, speaker identification, music generation. Apple Siri used LSTM-based models extensively in 2015–2019.
Time Series Forecasting
Stock price prediction, energy load forecasting, weather modelling, IoT sensor anomaly detection in manufacturing.
Robotics & Control
Autonomous driving (sequence of sensor readings), robotic arm control, reinforcement learning agents with temporal dependencies.

// TRAINING

06 Backpropagation Through Time

LSTMs are trained with Backpropagation Through Time (BPTT) — a generalisation of standard backprop where gradients are unrolled through the timestep dimension. The loss at each time step contributes gradients that flow back through all previous steps.

1
Forward pass: Run the LSTM over the full input sequence. Record all gate activations and states.
2
Compute loss: Compare output at each step (or the final step) to ground truth using cross-entropy, MSE, etc.
3
Backward pass: Unroll the computation graph. Gradients flow backwards through gates and cell states.
4
Truncated BPTT: For long sequences, gradients are often truncated after k steps to save memory and computation.
5
Update weights: Apply Adam, SGD, or RMSProp using accumulated gradients from all time steps.

Session Report & Export

Run a simulation, then preview and export your gate activations, states, and performance data. Follow the steps below to get the most from your export.

// HOW TO EXPORT — STEP BY STEP
1
Run a Simulation First
Navigate to the ⚡ Simulate tab. Either select a preset sequence (e.g. HELLO WORLD) or type your own custom input in the text box. Optionally upload a CSV/TXT dataset using the Upload Dataset mode, then click 🧠 Train & Simulate. Press ▶ Play or step through manually with ⏭ Step until the simulation completes all timesteps. The more steps you run, the richer your exported data will be.
2
Return to This Export Page
After running your simulation, click ⬇ Export in the navigation bar (or use the keyboard shortcut). The Session Report preview below will automatically populate with your simulation data — showing a table of every timestep with all four gate values, cell state averages, hidden state averages, and the memory retention label. If the preview shows "No simulation data yet", go back to Simulate and run at least one step.
3
Select What to Include
In the right-hand sidebar, use the checkboxes to customise your export content:
  • ◆Gate Activations — includes the forget (fₜ), input (iₜ), candidate (C̃ₜ), and output (oₜ) gate values for every timestep, each as a decimal between 0 and 1.
  • ◆State Vectors — includes the average cell state (long-term memory magnitude) and average hidden state (short-term output) values per step.
  • ◆Predictions — includes top-3 next-character probabilities derived from the hidden state's softmax distribution, useful for evaluating sequence learning quality.
  • ◆Session Parameters — includes the input sequence string, selected hidden size, vocabulary list, total steps run, and the timestamp of the session.
4
Choose Your Export Format
Two formats are available — choose based on your intended use:
⬇ Download .txt Report
A human-readable plain-text document. Opens in any text editor (Notepad, VS Code, TextEdit). Contains a formatted header with session metadata, followed by per-step blocks showing gate values and state summaries. Ideal for academic submission, presentations, and documentation. Filename: lstm_report.txt
⬇ Download .csv (per-step)
A comma-separated values file with one row per timestep. Column headers: Step, Char, Forget, Input, Candidate, Output, CellAvg, HiddenAvg, MemoryRetained. Opens directly in Excel, Google Sheets, or Python (pandas) for charting and further analysis. Filename: lstm_steps.csv
5
Click Download & Verify the File
Click the download button in the sidebar. Your browser will save the file to your default downloads folder automatically — no server upload involved, everything is generated locally in your browser. A green toast notification will confirm success. Open the file to verify all steps are present. The number of data rows in the CSV should match the sequence length shown in the simulation (e.g. "HELLO WORLD" = 11 rows). If the file appears empty, ensure you have run the simulation to completion before exporting.
6
Analyse & Re-Export as Needed
Each simulation session overwrites the previous data. To compare multiple sessions, download after each run before starting a new one. You can re-run the Simulate tab with a different sequence (e.g. switch from ABCABC to HELLO WORLD), then export again. For the dataset training mode, the exported CSV will also include training-derived gate values shaped by the LSTM weights fitted on your uploaded data, producing richer, more varied activations than the random-weight preset mode.
